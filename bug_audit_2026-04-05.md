## 🟢 Bugs corrigés (cette session)

### Critical — Backend crashes
- **[TIERS-010]** `NameError: current_user` dans `create_tier_contact` et `update_tier_contact` (les deux routes référençaient la variable sans la déclarer). → Fix : ajout du paramètre `current_user: User = Depends(get_current_user)`. Commit `fa83973`.
- **[TIERS-011]** `AttributeError: 'TierContactCreate' has no attribute 'email'` — le schéma Pydantic n'exposait pas `email`/`phone` alors que le modèle SQLAlchemy les a et que le code les référence. → Fix : ajout des champs dans `TierContactCreate` et `TierContactUpdate`. Commit `dd8faaf`.
- **[PROJECT-001]** `ProjectCreate.code` requis côté backend mais le formulaire disait "auto-généré" → 422 sur chaque création. → Fix : `code` rendu optionnel, auto-généré via `generate_reference("PRJ", ...)`. Commit `198a88e`.
- **[ALEMBIC-086]** Migration `086_ads_creator_and_initiator_review` crashait avec `DuplicateColumnError` sur `ads.created_by` (colonne déjà créée par un model auto-sync antérieur). → Fix : migration rendue idempotente (`ADD COLUMN IF NOT EXISTS`, DO-block pour la FK, `CREATE INDEX IF NOT EXISTS`). Commit `163b60f`.
- **[ALEMBIC-089]** Ma nouvelle migration 089 pointait vers `088_add_mission_visa_and_allowance_followups` qui n'est pas trackée dans git → `KeyError` en démarrage. → Fix : down_revision chaîné sur `086`. Commit `ba36345`.

### Critical — Frontend regressions
- **[TIERS-001]** Clés i18n `tiers.title`, `tiers.subtitle`, `tiers.create` affichées en brut sur la page Tiers. → **Root cause** : le fichier `fr/common.json` avait **deux blocs `tiers`** (ligne 828 et 2759), le parser JS utilise le dernier → les clés `title/subtitle/create` définies dans le premier étaient écrasées par le second. → Fix : merge automatique des clés dupliquées via `object_pairs_hook`. Commit `8781e34`.

### High — UX / Translation
- **[ACCENTS-*]** 30+ occurrences d'accents manquants corrigées dans le code et les traductions :
  - `Decouvrir → Découvrir`, `fonctionnalites → fonctionnalités`, `Priorite → Priorité`, `Planifie → Planifié`, `Termine → Terminé`, `Annule → Annulé`, `independant → indépendant`, `Helicoptere → Hélicoptère`, `Vehicule → Véhicule`, `DEPLACEMENT → DÉPLACEMENT`, `PLANIFIES → PLANIFIÉS`, `ARRIVES → ARRIVÉS`, `Meteo → Météo`.
  - Touchant : `CrossModuleLink.tsx`, `ConformitePage.tsx`, `DashboardPage.tsx`, `PidPfdPage.tsx`, `PlannerPage.tsx`, `ProjetsPage.tsx`, `TravelWizPage.tsx`, `fr/common.json`.
  - **Regression collatérale** introduite par la regex : `Annuler → Annulér`, `Terminer → Terminér`, `Planifier → Planifiér`, `Se connecter → Se connectér` (faux positifs sur les infinitifs). → Revert ciblé. Commit `af69e49`.
- **[TIERS-002/003/004]** Country code, language code, tier type affichés en brut (`CG`, `fr`, `client`). → Fix : `InlineEditableRow` supporte maintenant `displayValue`, `TierDetailPanel` résout via `useDictionaryLabels('country')` + `Intl.DisplayNames` pour les langues.
- **[TIERS-005]** Obligations légales dupliquées (`RCCM, NIU, SIRET, SIREN, NIF, RCCM, RCCM, NIF, NINEA, RCCM`). → Fix : dédup via `new Set(...)` dans `LegalIdentifierManager`.
- **[TIERS-007]** Mois de début d'exercice affiché en chiffre brut (`1`). → Fix : `Intl.DateTimeFormat({month: 'long'})` → "Janvier".
- **[TIERS-008]** `normalizeNames()` uppercasait tous les noms → perte d'information (`François` → `FRANCOIS`). → Fix : trim + collapse whitespace, preserve la casse. Commit `8781e34`.

### High — Security / CORS
- **[CORS-500]** Une exception 500 non catchée dans FastAPI contourne le `CORSMiddleware`, le navigateur voit une erreur CORS trompeuse au lieu du vrai 500. → Fix : ajout d'un `@app.exception_handler(Exception)` global qui retourne un JSON 500 avec les headers CORS. Commit `8781e34`.

### Medium — Backend data integrity
- **[ALEMBIC-089]** Nouvelle migration qui :
  - Ajoute `archived` + `deleted_at` IF NOT EXISTS sur `transport_vectors` et `voyages` (SoftDeleteMixin déclaré sur le modèle mais aucune migration n'avait été générée → chaque requête TravelWiz échouait avec `UndefinedColumnError`).
  - Dédup les settings dupliqués puis ajoute `UNIQUE (key, scope)` sur `public.settings` (6 lignes dupliquées dans `integration.gouti.*` avant dédup).
  - Seed 35 entrées de dictionnaires manquantes : `visit_category`, `transport_mode`, `civility`, `tier_type`, `mission_type`, `mission_activity_type`, `pax_type`.
  - Commit `8781e34`.

### High — Dashboard / Stats

#### [DASH-001] Cartes de stats toutes à 0 (2 tiers, 1 projet existants)
- **Page** : `/dashboard`
- **Symptôme** : Les widgets "Assets actifs", "Utilisateurs", "Entreprises" affichent `0` alors que la DB contient 2 entreprises, 1 projet et plusieurs utilisateurs.
- **Cause probable** : L'endpoint `/api/v1/dashboard/widgets/stats` ne compte pas correctement (caché ou query filtrée par tenant alors que tout est en `public`).

#### [DASH-003] Activity feed affiche des clés i18n brutes
- Exemples : `dashboard.action_paxlog.avm.cancel mission_notice 931be72f-...`, `dashboard.action_paxlog.external_link.create external_access_link 20e0e1dc-...`

- **Fix** : Ajouter `dashboard.action_paxlog.*` en `fr/en common.json`.

- `Connexion user 2eca347d-f502-4f39-9b35-01eb45665560 il y a 1 seconde`

#### [DASH-002] Accord de genre sur "Entreprises ... actifs"
