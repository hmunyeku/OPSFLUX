PLATEFORME · ARCHITECTURE GÉNÉRALE

OpsFlux — l'application

SPA React / API FastAPI, modulaire et multi-filiale, avec un cadre de navigation unifié pour 15+ modules métier.
  • Architecture modulaire : chaque métier est un module activable / désactivable par filiale
  • Contrôle d'accès fin (RBAC) : permissions par module, garde sur chaque écran
  • Multi-tenant : cloisonnement des données par entité / filiale
  • Authentification : login, MFA, vérification e-mail, SSO / Active Directory
  • Interface unifiée : barre latérale, panneau de détail, recherche, notifications temps réel (WebSocket)
  • Aide intégrée + assistant avec capture de tickets
Objets gérés : Utilisateurs · Entités · Modules · Rôles · Permissions · Paramètres

MODULE · GESTION DE PROJET

Projets

Cycle de vie projet complet : tâches, jalons, livrables, révisions de planning et Gantt avec chemin critique.
  • Hiérarchie de tâches (WBS) avec dépendances prédécesseur / successeur
  • Gantt avec calcul du chemin critique (CPM) et marges
  • Révisions de planning : snapshot, comparaison, application
  • Jalons avec consolidation de statut
  • Livrables avec circuit d'acceptation / rejet
  • Intégration Planner : bascule des tâches vers l'ordonnanceur
Objets gérés : Projet · Tâche · Jalon · Livrable · Révision de planning

MODULE · ORDONNANCEMENT

Planner

Ordonnanceur multi-sites des activités, avec détection de conflits et visualisation de la capacité.
  • Types d'activités : projet, workover, forage, maintenance, inspection, opérations permanentes
  • Détection de conflits : chevauchements sur un même actif / puits, arbitrage par priorité
  • Heatmap de capacité : seuils faible / moyen / élevé / critique configurables
  • Workflow de validation : brouillon → soumis → validé → en cours → terminé
  • Dépendances et contraintes d'ordonnancement
  • Procédure d'urgence (override maintenance)
Objets gérés : Activité · Conflit · Scénario · Dépendance

MODULE · MANAGEMENT OF CHANGE

MOCTrack

Digitalisation de la gestion du changement : workflow à 11 statuts, matrice de validation multi-rôles et signatures.
  • Workflow à 11 statuts, de la création à la clôture documentaire
  • Matrice de validation parallèle (HSE, Maintenance, Process, Production, Métier) avec visas signés
  • Type (permanent / temporaire), nature (optimisation / sécurité), priorité et imputation des coûts
  • Clôture documentaire PID / ESD distincte de l'exécution (visa chef de site)
  • Promotion d'un MOC en Projet
  • Notifications temps réel à chaque transition d'état

MODULE · TRANSPORT & LOGISTIQUE PASSAGERS

TravelWiz

Coordination de bout en bout du transport : vecteurs, voyages, manifestes passagers, plans de pont et portail capitaine.
