Conçu pour les opérateurs industriels exigeants

Les défis terrain (sites isolés, conformité, traçabilité réglementaire, multi-pays) sont au cœur du design produit.

    Champs onshore, plateformes offshore, FPSO. Rotations, embarquement, conformité réglementaire.

  • Maritime

    Affrètements, manifestes passagers, captain portal, suivi cargo public via QR code.

  • Énergie

    Centrales, parcs solaires, fermes éoliennes. Maintenance préventive, asset registry, rapports.

  • Mining

    Sites isolés, cycles d'équipes, tonnage, blast permits, traçabilité matériel.

  • Construction

    EPC offshore, lots multi-corps, sous-traitants, contrôles d'accès et habilitations.

    Architecture & sécurité

    Une stack open source moderne, conçue pour durer.

    01

    Multi-tenant strict

    Schemas PostgreSQL par tenant, scoping entity_id sur toutes les tables métier. Aucune fuite cross-tenant possible — les middlewares forcent le filtre à chaque requête SQL.

    02

    Auth & RBAC granulaire

    SSO Google, Microsoft, GitHub, Okta, Keycloak. JWT + refresh. Permissions hiérarchiques par module (~500), rôles configurables, audit log complet.

    03

    Stack technique

    Python 3.12 / FastAPI / SQLAlchemy 2 async, PostgreSQL 16 (pgvector + PostGIS + ltree), Redis 7, React 18 / Vite / TailwindCSS / shadcn-ui. Mobile React Native (Expo).

    04

    Self-hostable

    Docker Compose sur n'importe quel VPS — vanilla, Dokploy, Coolify, EasyPanel, Caprover, Portainer. Traefik + Let's Encrypt. Variables DOMAIN, STACK_NAME, CERT_RESOLVER, TRAEFIK_NETWORK.

    Comment démarrer

    Trois étapes pour mettre OpsFlux en production.

    1. 1

      Configurez votre domaine

      Pointez 8 sous-domaines (app., api., docs., ext., drawio., db., www., apex) sur l'IP de votre VPS. Ajoutez un wildcard si vous avez un cert.

    2. 2

      Lancez Docker Compose

      cp .env.example .env, complétez POSTGRES_PASSWORD, SECRET_KEY, JWT_SECRET_KEY, ENCRYPTION_KEY, puis docker compose up -d. Migrations + seed automatiques au boot.

    3. 3

      Connectez-vous

      Login admin via FIRST_SUPERUSER. Activez les modules, configurez les SSO providers depuis Settings → Intégrations, invitez vos premiers utilisateurs.
